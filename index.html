01

NOW · live session

Project, model, context %, spend, burn rate, cache savings — updated every time a JSONL line lands. The only group that ticks between breaths.

02

TODAY · aggregated

Rolling window since midnight local. Tokens, cached %, session count, longest session, and which project ate the most of your day.

03

QUOTA · OAuth (optional)

The only group that calls out. Opt-in via toksight.oauthEnabled to pull the 5h and 7d windows Anthropic actually counts against — plus the Sonnet 7-day sub-window.

04

MODELS & TOOLS · 7d

Model mix by cost and token share. Tool call counts with error rates — so you notice when Bash has been failing half the time.

05

INSIGHTS · rule-based

Up to 3 actionable observations from the insights engine: cache wins, friction points, unusual patterns. No LLM round-trip — just deterministic rules over your own data.

Local, by design

Nothing leaves your machine.

TokSight is a read-only IDE extension. It watches ~/.claude/projects/ for changes, parses them in memory, and renders the result. There is no backend, no analytics, no "anonymous" telemetry, no account.

  • ✓ Zero network calls (except optional OAuth you opt into)
  • ✓ No CLAUDE.md content ever parsed or sent
  • ✓ Every surface hides cleanly when its source is missing
  • ✓ MIT licensed, source on GitHub
  • ✓ Disable at any time — your data never leaves the machine
Questions

Before you install.

Is TokSight free?+
Yes. Completely free, MIT-licensed, and source is on GitHub. No pro tier, no upsell.
Does TokSight work with Enterprise Claude Code?+
Yes. TokSight reads directly from local JSONL session files. It works with any Claude Code deployment — Anthropic, Vertex, Bedrock, Enterprise — as long as your CLI writes to the usual ~/.claude/projects/ directory.
How is this different from the /usage command?+
The CLI's /usage shows you a snapshot for the current session. TokSight gives you an always-on widget, full session history, per-project splits, quota windows, model breakdowns, and the whole thing lives inside your IDE.
Does TokSight collect any data?+
No. Zero network, zero telemetry. The only outbound request is the optional Anthropic OAuth quota call you have to explicitly enable.
Will the numbers match claude /cost?+
Yes, to the cent. TokSight uses the same pricing tables and the same source files the CLI writes. If they diverge, it's a bug — please file it.
What does the optional OAuth get me?+
The official 5-hour and 7-day utilization numbers Anthropic reports — plus the Sonnet 7-day sub-window. Without OAuth, TokSight still estimates cost from local data; with OAuth, the quota bars show exact utilization. Toggle via toksight.oauthEnabled.
